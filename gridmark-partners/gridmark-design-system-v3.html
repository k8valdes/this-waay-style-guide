01 · Principles

Design principles

Six commitments inherited from This Waay and grounded for civic work. They decide the close calls the tokens can't.

Evidence over hype

Research and a clear reference point before tools. The system favors clarity over decoration.

Structure before scale

Governance, rhythm, and tokens come first so the system grows without rework.

Human-centered throughout

Every component is judged by whether it works for residents and the staff who run the service.

Calm authority

One accent, generous space, no shout. Confidence shown through restraint.

Accessible by default

AA contrast, visible focus, keyboard paths, and reduced-motion are baseline, not add-ons.

Sibling, not stranger

70% shared with This Waay; the civic 30% is expressed through structure, not a new palette.

02 · Architecture

Token layers

Three layers. Primitives hold raw brand values; semantic tokens give them meaning and are what components reference; component tokens tune individual parts. The site never hardcodes a hex — it consumes semantic tokens, so a theme change is a token change.

/* L1 primitive → L2 semantic → L3 component */ --navy-900: #0C2A48; /* brand primitive */ --green-500: #54B987; /* brand primitive */ --text-primary: var(--navy-900); /* semantic alias */ --action: var(--green-500); /* semantic alias */ --action-hover: var(--green-600); /* derived state */ --btn-primary-bg: var(--action); /* component token */
Naming convention: --[category]-[role]-[variant?]-[state?]. Categories: color/text/bg/border, space, radius, elev, dur, ease, z, font. Derived steps (states, status, neutrals) are tuned to the palette and AA — they are never new brand colors.

08 · Foundations

Accessibility commitment

Gridmark's market explicitly buys accessibility — California prioritizes Section 508 / ADA and digital equity. The system targets WCAG 2.1 AA as a floor, and accessibility is a sellable service, so the site must model it.

Built in

  • AA contrast on all text (table in §03)
  • Visible focus ring on every interactive element (--focus-ring)
  • Full keyboard operability; logical tab order
  • Targets ≥ 44×44px; ≥ 24px spacing
  • prefers-reduced-motion honored globally
  • Semantic HTML, ARIA only to fill gaps

Never rely on

  • Color alone to convey state (pair an icon/label)
  • Green text on white for anything readable
  • Placeholder text as a label
  • Hover-only disclosure of essential info
  • Motion that conveys meaning without a static fallback

11 · Application

Page patterns

The marketing site is assembled from these blocks. Every page follows the This Waay archetype: a navy hero, then alternating navy and soft-gradient sections on shallow angled breaks, each opening with a centered eyebrow → H2 → lede lockup.

/* page archetype */ 1 Navy hero — eyebrow · headline (one green highlight) · lede · primary + secondary CTA 2 Soft section — centered header lockup, then card grid (badge or content cards) 3 Navy section — angled break · "Find the signal / Build the system" methodology 4 Soft section — "Who we serve" audience cards 5 Navy section — procurement readiness · NAICS/UNSPSC/PSC data block 6 Soft section — testimonial / past-performance panel cards 7 Navy footer — contact · endorsement lockup · nav

12 · Language

Voice & UI writing

Composed, observational, structural — This Waay's voice, grounded for civic work. Sentence case, plain verbs, specificity over volume.

Sounds like us
  • "We didn't add AI. We prepared the system for it."
  • "Operational clarity. Governed intelligence."
  • "A right-sized first step that earns the next phase."
  • Buttons say what happens: "Request a capability statement."
Never
  • Hype: "revolutionary," "game-changing," "supercharge"
  • "Users / customers" → say "residents," "the public"
  • "AI-first" → "responsible AI," "AI with purpose"
  • Exclamation points in professional contexts
13 · Operations

Governance & versioning

A living system. Semantic versioning, a changelog, and clear rules for how AI tools and designers consume the tokens keep it from drifting.

How AI tools should use this system

  1. Reference semantic tokens, never raw hex from memory.
  2. One green for actions/labels/accents; brass at most once per asset.
  3. Sentence-case headlines, one highlight max; eyebrows are the only caps.
  4. Axiforma, Poppins fallback — never a second family, never a serif.
  5. Frame screenshots; annotate only in green; coral stays in illustration.
  6. Check contrast against §03 before placing text on color.
